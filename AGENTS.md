# AGENTS.md

Guidance for AI coding agents and maintainers working in this repository.

## Mission and tone

This project helps Christian technologists, designers, publishers, churches, and
builders download, share, adapt, and build creative Scripture resources around
the public-domain Berean Standard Bible (BSB).

Keep changes aligned with that goal: make Scripture easier to distribute,
study, print, link, read, or build with. This is an unofficial community toolkit,
not an official Berean Bible project.

## Public-repo hygiene

- Do not commit secrets, API keys, wallets, local `.env` files, personal notes,
  agent memory, screenshots, temporary scripts, or work-in-progress artifacts.
- Keep `.agents/`, local work PDFs, source archives, visual QA images, build
  outputs, and package metadata generated by tools out of git unless a tracked
  exception is explicitly documented.
- The current committed PDF exceptions are:
  - `drafts/primary/bsb-primary-draft.pdf`
  - `drafts/primary/bsb-single-column-draft.pdf`
- If you add third-party assets, document their license and attribution in
  `NOTICE` or a nearby license file.

## BSB text and naming

- The BSB text is public domain / CC0 according to the official Berean terms.
- If generated resources keep the BSB text verbatim, it is appropriate to refer
  to them as BSB-based resources.
- If you intentionally alter the biblical text, document that clearly and do not
  present the result as an official Berean Bible text.

## Development setup

Prefer an isolated environment:

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

For console scripts:

```bash
python -m pip install -e .
```

Use `python3` instead of `python` on systems where `python` is unavailable.

## Verification

For code or release-artifact changes, run the smallest relevant check first, then
broaden if needed.

Recommended public-readiness checks:

```bash
python -m compileall -q src scripts design_bsb.py smoke_test.py setup.py
python design_bsb.py --qa-only --verify
git diff --check
```

If changing single-column layout behavior, regenerate the affected draft and run
the smoke test:

```bash
PYTHONPATH=src python -m bsb_pdf_toolkit.generate_reflow_pdf \
  drafts/primary/source/engbsb_usfm.zip \
  drafts/primary/bsb-single-column-draft.pdf \
  --font-dir fonts --columns 1
python smoke_test.py
```

If changing `.github/workflows/deliver-assets.yml`, parse or lint the workflow
before finishing when tooling is available.

## Generated artifacts and manifests

When a committed PDF artifact changes, update `drafts/primary/README.md` with:

- page counts;
- link counts;
- file sizes;
- SHA-256 hashes;
- semantic fingerprints;
- any relevant layout knobs or source notes.

Do not update expected verifier constants by guessing. Inspect the generated
artifacts and keep the manifest, verifier, and committed PDFs consistent.

## Release workflow safety

Publishing to GitHub Releases, itch.io, or Permaweb should remain manual and
intentional. Keep dry runs safe by default, and do not make normal pushes publish
external assets.

## Code style

- Prefer small, direct changes that match the existing Python style.
- Avoid unrelated refactors while preparing release artifacts.
- Keep CLI behavior documented in `README.md` when it affects public users.
- Prefer reproducible commands over hidden local state.
